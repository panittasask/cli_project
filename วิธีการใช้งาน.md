# วิธีการใช้งาน Local llama.cpp Server

ระบบนี้แบ่งเป็น 2 เครื่อง:

- **เครื่อง Server** เก็บ GGUF และใช้ GPU รัน `llama-server`
- **เครื่อง Office** รัน CLI และส่งคำขอหา Server ผ่าน Tailscale

พอร์ต 8080 ไม่ได้เปิดสู่ Internet โดยตรง การเชื่อมต่อใช้ Tailscale tailnet เท่านั้น

## 1. ติดตั้ง Server ให้เปิดอัตโนมัติ

ทำครั้งเดียวบนเครื่อง Server:

```powershell
cd G:\cli_project
npm install
npm run server:install
```

Windows จะแสดง UAC ให้กด **Yes** คำสั่งจะสร้าง Scheduled Task ชื่อ
`LocalLlamaServer` ซึ่ง:

- รันด้วยบัญชี `SYSTEM` ตอน Windows startup
- เปิด llama.cpp router จาก `G:\Model`
- โหลด `defaultModel` จาก `.cli/settings.json` อัตโนมัติ
- โหลดพร้อมกันสูงสุดตาม `modelsMax` (เครื่องนี้ตั้งไว้ 1)
- restart อัตโนมัติภายในประมาณ 1 นาทีหาก process ล้ม
- ไม่ต้องเปิด Terminal หรือ remote เข้าเครื่องหลัง reboot

ตรวจสถานะ:

```powershell
npm run server:status
```

Log อยู่ที่:

```text
G:\cli_project\.cli\logs\llama-autostart.log
```

## 2. ตั้งค่าเครื่อง Office

ติดตั้ง Tailscale และ login เข้า tailnet บัญชีเดียวกับ Server ก่อน จากนั้น:

```powershell
git pull
npm install
setx LLAMA_API_URL "http://100.115.8.124:8080/v1/chat/completions"
$env:LLAMA_API_URL = "http://100.115.8.124:8080/v1/chat/completions"
npm run dev:cli
```

`setx` เก็บค่าสำหรับ Terminal ครั้งถัดไป ส่วน `$env:` ทำให้ Terminal ปัจจุบันใช้ค่าได้ทันที

ครั้งต่อไปเปิดเพียง:

```powershell
cd <ตำแหน่ง repo บนเครื่อง Office>
npm run dev:cli
```

เครื่อง Office ไม่ต้องมี llama.cpp, GGUF หรือ GPU สำหรับ inference

## 3. ตรวจการเชื่อมต่อจาก Office

```powershell
tailscale ping 100.115.8.124
Invoke-RestMethod http://100.115.8.124:8080/health
Invoke-RestMethod http://100.115.8.124:8080/models
```

Health ที่ถูกต้องคือ:

```text
status : ok
```

ตอน CLI เริ่มทำงานควรแสดง API และโมเดลจาก Server ไม่ใช่ configured fallback

## 4. ดูและเปลี่ยนโมเดลจาก Office

ภายใน CLI:

```text
/model
/model 2
/model Qwen3-14B-Q4_K_M
```

`/model` แสดงโมเดลทั้งหมดพร้อมสถานะ `loaded`/`unloaded` การสลับจะ unload
โมเดลเดิมก่อน โหลดโมเดลใหม่ และล้าง active task context แต่ไม่ลบ session history

## 5. คำสั่งดูแล Server

ตรวจสถานะ:

```powershell
npm run server:status
```

Restart หลังแก้ settings หรือ pull code ใหม่:

```powershell
npm run server:restart
```

ติดตั้ง/อัปเดต Scheduled Task ซ้ำ:

```powershell
npm run server:install
```

ยกเลิก autostart และหยุด llama-server:

```powershell
npm run server:uninstall
```

คำสั่ง install, restart และ uninstall ต้องยืนยัน UAC

## 6. หลังอัปเดต repo บน Server

Scheduled Task ชี้มาที่ repo นี้โดยตรง หลัง `git pull` ให้ restart หนึ่งครั้ง:

```powershell
cd G:\cli_project
git pull
npm install
npm run server:restart
```

## 7. แก้ปัญหาเบื้องต้น

ถ้า Office แสดง `Current model: unavailable`:

```powershell
$env:LLAMA_API_URL
tailscale status
tailscale ping 100.115.8.124
Invoke-RestMethod http://100.115.8.124:8080/health
```

ถ้า Server ไม่ตอบ:

```powershell
npm run server:status
Get-Content .cli\logs\llama-autostart.log -Tail 100
npm run server:restart
```

ถ้า IP ของ Tailscale เปลี่ยน ให้ดูค่าปัจจุบันบน Server ด้วย:

```powershell
tailscale ip -4
```
